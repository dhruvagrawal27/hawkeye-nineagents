# HAWKEYE — 5-minute judge walkthrough

This is the script. Open the live URL, do not improvise — every step has a known-good outcome.

## Pre-flight (once, before judges arrive)

```bash
ssh root@<vps-ip>
cd /opt/hawkeye
docker compose ps                  # all 12 services should be Up (healthy)
docker compose exec -T backend python -m app.scripts.preflight_check
# Expected: all 12 checks PASS, exits 0
```

If preflight fails, do **not** demo. Run `make seed` again, then preflight.

## Step 1 — Login (30s)

Open https://hawkeye.nineagents.in. Login with `analyst@hawkeye.local` / `analyst`.

**Expected**: Redirected to Keycloak (clean dark login screen), back to dashboard.

**What to say**: *"HAWKEYE is multi-tenant. Analysts and supervisors get role-scoped views. Auth is OIDC via Keycloak — meets RBI ITV-2 norms for SSO + audit trail."*

## Step 2 — Dashboard immediately populated (30s)

**Expected**: Dashboard loads with 50 seeded alerts, 4 stat cards already populated, risk-distribution histogram, alert heatmap.

**What to say**: *"On any boot, HAWKEYE pre-computes the top 50 highest-risk employees from the trained model and surfaces them as alerts. The dashboard is never empty — even if Kafka is down, the analyst has work to do."*

## Step 3 — Drill into one alert (60s)

Click the topmost CRITICAL alert. Slide-over panel opens. Click "Open employee detail."

**Expected**: Employee detail page with 5 tabs. Header shows large animated risk gauge (~0.85+ display score), department, alerts-this-week count.

Click **SHAP Analysis** tab. Show the waterfall chart. Hover the longest bar.

**What to say**: *"Each prediction is fully auditable. SHAP values explain why the model thinks this employee is risky — pass-through rate of 97%, structuring activity 17×, off-hours fraction 69%. RBI FREE-AI requires a human readable factor breakdown for any AI-driven action; this is it."*

Click **Narrative** tab. Click "Generate narrative" if not yet generated.

**Expected**: 4-paragraph memo (Risk Summary / Observed / Why It Matters / Recommended Next Step) plus a collapsible "Audit trail — model factors" footer with the raw SHAP contributions.

**What to say**: *"Generated by an LLM (Groq, model openai/gpt-oss-120b), grounded on the SHAP factors. The audit footer is non-removable — every memo includes the model factors for compliance review. If the LLM is unreachable, we degrade to a deterministic fallback so the workflow never blocks."*

## Step 4 — Live replay → alerts fire on screen (90s)

Navigate to **Replay Studio** (sidebar). Click big **▶ Start Replay** (mule_burst, 500 ev/s).

**Expected**: Within 10–60 seconds, 3+ new alerts appear in the AlertFeed on the right. The events/sec sparkline rises. Score-distribution mini-histogram updates.

Click the prominent **"Inject Mule Burst"** button.

**Expected**: 3+ additional alerts within 10 seconds, with toast notifications.

**What to say**: *"This is the production data path: synthetic events flow through Kafka into the consumer, which updates per-account features in Redis and re-scores. When the blended LightGBM score crosses threshold, the alert is created, an LLM narrative is generated, and the WebSocket pushes it to every connected analyst."*

Stop replay (■ Stop). Close.

## Step 5 — Graph explorer (60s)

Navigate to **Graph Explorer**. Click **Collusion Clusters**.

**Expected**: Force-directed graph renders. Distinct colour clusters appear after community-detection runs.

Search for an employee id from one of the alerts (e.g., `EMP_062946`). Camera flies to it.

**What to say**: *"Insider fraud rings often share systems. The graph layer joins the dots — every employee linked to every system they touch, every shared workstation, every common IP. Communities surface naturally; the analyst doesn't have to write Cypher queries."*

## Step 6 — Settings & compliance (30s)

Navigate to **Settings**.

**Expected**: Model Card shows AUC 0.998, F1 0.967, threshold 0.16. System health shows all five services green. Compliance footer at the bottom.

**What to say**: *"Model card surfaces lineage and metrics. System health is live. The compliance footer is the human-in-the-loop disclaimer — required by RBI FREE-AI."*

## If something breaks mid-demo

| Symptom | Recover |
|---|---|
| Dashboard empty | `ssh` + `docker compose exec backend python -m app.scripts.seed` |
| Replay button does nothing | `docker compose restart backend kafka` then re-try replay |
| Narrative panel hangs | The Groq fallback should have kicked in — refresh the page; if still hung, `docker compose logs backend \| grep narrative` |
| Graph blank | `docker compose restart neo4j backend` and wait 20s |

Always have `docker compose logs -f backend` in a second terminal during the demo.
